Introduction

Sun Application Server 8.1 and Eclipse 3.1 RC and WPT:

The new Eclipse (3.1) will be available at JavaOne 2005. It is used as the basis for the Web Tool Platform (WTP) project which is supposed to deliver J2EE support on top of Eclipse. BEA, Oracle, JBOSS, and IBM are now the key owners of the WTP area.

WTP is composed of 2 sub projects:

  • WEB Standard Tools (WST): http://www.eclipse.org/webtools/wst/main.html
  • J2EE Standard Tools (JST): http://www.eclipse.org/webtools/jst/main.html

The WTP project has now a target date of December 2005. So far, WTP has simple plugins for JBoss, Tomcat, Geronimo, JONA, and Weblogic 8.1.

WTP 1.0 (and even 1.1) will  not support the entire J2EE 1.4 specifications, and will not support the new Ease of Development features introduced in the Java Enterprise Edition 5 (EJB 3.0 annotations,...). For example, there is no plan to support at all EJB CMP creation or mapping. . Also, The Eclipse  IDE does not support JSR 88 so that we cannot reuse all the work done on the Glassfish Application Server side or even in  the NetBeans IDE  side for Sun specific deployment descriptors editing, or J2EE application deployment.

Eclipse Plugin: a prototype for Sun Application Server 8.1 and Glassfish

The best way to learn how to write an Eclipse plugin is actually to write one. A prototype plugin is now available that exposes the following features:

  • Register the Sun Application Server or the Glassfish Server
  • Select the Server options
  • Add/Remove Eclipse projects to this Server
  • display Server Status, and Project Explorer showing the J2EE APIS added to the project.
  • Start/stop the Application Server from Eclipse

Configuring and Using GlassFish Server from Eclipse IDE

To create a new GlassFish Server

The GlassFish Tools Bundle for Eclipse will install a new instance of GlassFish v2.1 Server and GlassFish v3 Prelude for you automatically. If you need to create another new instance of the GlassFish Server, use the following procedure.

  1. Select Servers tab.
  2. From GlassFish Server popup menu, select New.
  3. From New Server page, complete the required parameters:

    Server's host name

    Server type

    Server name

    Server Runtime Environment

  4. Click Next to complete Server configuration details.
  5. Click Next to see Add and Remove Projects page.

    Select any projects you want to add to the new GlassFish Server

  6. Click Finish to complete the new Server creation.

To start GlassFish Server

This task describes the process of starting your GlassFish Server after installation.

Note - By default, Eclipse Workbench is set to Java EE Perspective. Skip Step 1 if the Workbench is already set to Java EE Perspective.

Before You Begin

Make sure the Eclipse IDE is open.

  1. Set the Eclipse IDE to Java EE Perspective, by selecting Window->Open Perspective->Java EE Perspective.

    Eclipse IDE is set to Java EE Perspective.

  2. Select the GlassFish Server instance you want to start, from Servers tab menu.
  3. Start the GlassFish Server from Server Popup menu.

    Note - You can learn more about how to access GlassFish Server Popup menu from Appendix A, Using Popup Menu

To stop GlassFish Server

To stop the running GlassFish Server from Eclipse IDE, use the following procedure.

  1. Select Servers tab.
  2. Select the GlassFish Server you want to stop.
  3. From GlassFish Server Popup menu, select Stop.

    GlassFish Server is started.

To restart GlassFish Server

To stop and restart the GlassFish Server, use the following procedure.

  1. Select Server tab.
  2. Select the GlassFish Server you want to restart.
  3. From GlassFish Server Popup menu, select Restart.

    GlassFish Server is stopped and restarted.

To start GlassFish Server in debug mode

When you want to debug an application, run the GlassFish Server in debug mode. To start the GlassFish Server in debug mode use the following procedure.

  1. Select Servers tab.
  2. Select the GlassFish Server, you want to start in debug mode.
  3. From GlassFish Server Popup menu, select Debug.

    GlassFish Server is in Debug mode.

    Note - If the GlassFish Server is already running, you will see the Restart in Debug Menu option.

    You can learn more about using the GlassFish Server to debug the applications from here:To debug a Servlet with the GlassFish Server

To publish Projects

You can publish the applications to the GlassFish Server, by using the following procedure.

  1. Select Servers tab.
  2. Select the GlassFish Server.
  3. From the GlassFish Server Popup menu, select Publish.

    The applications are published to the GlassFish Server.

To clean published Projects

To discard all the published projects, and republish from scratch, use the following procedure.

  1. Select Servers tab.
  2. Select the GlassFish Server.
  3. From the GlassFish Server Popup menu, select Clean.
  4. Click OK to accept the warning regarding discarding the published state.

    The published state of projects is discarded and republished.

To add and remove Projects

You can add and remove the projects that are configured to run on GlassFish Server with the following procedure.

  1. Select Server tab.
  2. Select the GlassFish Server.
  3. From GlassFish Server Popup menu, select Add and Remove Projects.

    Add and Remove Project screen is displayed.

  4. To add a project, select the project you want to add to the Server from the Available projects panel.
  5. Click Add, to add the project to the Configured projects panel.
  6. To remove a project, select the project you want to remove from the Configured projects panel.
  7. Click Remove, to remove the project to the Available projects panel.
  8. Click Finish to exit..

To display GlassFish Sever properties

To view some of the GlassFish Server current properties and to restore them to default, use the following procedure.

  2. Select Server tab.
  3. Select the GlassFish Server.
  4. From GlassFish Server Popup menu, select Properties.

    Properties for GlassFish Server screen is displayed.

  5. View the current properties or restore them to default.

To access GlassFish Server Administration Console

The GlassFish Sever offers many administrative features that are not accessible from the Eclipse IDE. These features are available from GlassFish Server Administration Console which is accessible from Eclipse IDE. The following steps describe how to access GlassFish Server Administration Console from Eclipse IDE.

Before You Begin

Make sure that the GlassFish Server is up and running.

  1. Open the Eclipse IDE.
  2. Select Servers tab.
  3. Select GlassFish Server Popup menu -> GlassFish Enterprise Server ->GlassFish Administration Console.

    GlassFish Administration Console is displayed.

    Note - You can learn more about how to access GlassFish Server Popup menu from Appendix A, Using Popup Menu

  4. Access various features available from the Administration Console such as Registration, Subscription and Documentation.

To register GlassFish Server Installation

By registering your GlassFish Server installation, you can receive product support, updates and community news. If you did not register during installation, use the following procedure to register your GlassFish Server from Eclipse IDE.

Before You Begin

Make sure that the GlassFish Server is up and running. You can learn more about how to access GlassFish Server Popup menu from Appendix A, Using Popup Menu

  1. Select GlassFish Server Popup menu ->GlassFish Enterprise Server ->Register GlassFish Enterprise Server to view the Registration page
  2. Provide the required information to register the GlassFish Server Installation.
  3. To access the Support Information, select GlassFish Server Popup menu ->GlassFish Enterprise Server ->Get Support for GlassFish Enterprise Server to view the Support page

To use GlassFish Update Center from Eclipse IDE

By using the GlassFish Update Center you can access Add-On components and product updates for GlassFish Server. Use the following procedure to access the GlassFish Update Center from the Eclipse IDE.

Before You Begin

Make sure that the GlassFish Server is up and running. You can learn more about how to access GlassFish Server Popup menu from Appendix A, Using Popup Menu

  1. Select GlassFish Server Popup menu ->GlassFish Enterprise Server ->GlassFish Update Center to view the Update Center
  2. Select Add-On components and product updates as required.

To use Help menu for GlassFish Server

In the GlassFish Tools Bundle for Eclipse, additional GlassFish help options are added to the Help menu. You can access the Online Help for GlassFish Server v2.1 and GlassFish Server v3 Prelude from Eclipse IDE Help->GlassFish Enterprise Server menu. The Help options are useful for learning how to use, administer or deploy with GlassFish server.

  1. To view Java EE javadoc, select Help->Java EE 5 Javadoc.
  2. To view GlassFish Server help options, select Help->GlassFish Enterprise Server.

    The following options are available:

    • Get Support for GlassFish Enterprise Server
    • Aquarium: GlassFish Community News
    • GlassFish v3 Prelude (Documentation)
    • GlassFish v2.1 Java EE 5 (Documentation)

To change Preferences for GlassFish Server

The GlassFish Tools Bundle for Eclipse allows you to change the Global preferences for GlassFish Server from Eclipse IDE.

  • To change GlassFish Server Global preferences from Eclipse IDE, select Window->Preferences.

    The following Global Preferences options are explained in detail.

    • Enable GlassFish Plug-in Log information in IDE log file

      This preference is useful for the debugging purposes. Turning on this option allows you to see additional traces in the Eclipse log file specific to the GlassFish Plug-in. This option is OFF by default.

    • Start the JavaDB database process when Starting GlassFish Server

      GlassFish Tools Bundle for Eclipse includes a preconfigured JavaDB Database. This option allows you to start the database automatically when the server is started. It is useful when working with JPA applications. This option is ON by default.

    • Start the GlassFish Enterprise Server in Verbose Mode

      Turning on this option will send all the output log of the GlassFish server to the Eclipse console. This option is OFF by default as there is a GlassFish specific log view.

    • Enable colored rendering in the GlassFish Log Viewer

      This option will enable messages to be displayed in color. INFO level messages will be in black, WARNING messages will be in yellow, SEVERE messages will be in red, and multiple line entries will be in Italic. This option is ON by default.

To monitor TCP/IP for Web Applications with GlassFish Server

TCP/IP monitoring is a useful Eclipse IDE feature that is enabled for the GlassFish Server in this bundle. It allows you to trace HTTP requests made to your web application. The procedure to configure the TCP/IP monitoring is as follows:

  1. Select the GlassFish Server Popup menu Monitoring->Properties.

    Monitoring Properties page is displayed.

  2. Select the Add button on the right side of panel.

    Monitoring Ports page is displayed.

  3. Select the Server Port, which is the http port used by the GlassFish server. By default, it is 8080. The port that will be used to actually do the monitoring is 8081 as specified in the Monitor Port field.

    You can modify the port number as required.

  4. To start monitoring, press the Start button on the right side. The port 8080 is now monitored from port 8081.

    The Status column in the page will show status as Started.

  5. Select the Apply button and then the OK button, to complete the setup.

    When the GlassFish server is started, access a web page using the port 8081 instead of the regular port 8080. The TCP/IP monitor view should appear below, and fill up with the traffic between your Web browser and your web application. When you select each request in the tree at the top left, information about the request/response is shown, the bottom left of the view will display the HTTP request that was made to your web application, and the bottom right will show each response.

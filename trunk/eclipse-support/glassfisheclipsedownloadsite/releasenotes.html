What's New in 1.0

This section provides an overview of the product and what is new in this release.

The GlassFish Tools Bundle for Eclipse is an open source project. It is a co-bundle of Eclipse IDE, GlassFish Server v2.1 and GlassFish Server v3 Prelude and the plug-in that integrates these products.

The GlassFish Tools Bundle for Eclipse allows you to build applications with the Eclipse IDE and deploy them to the GlassFish Server.

The following new features are available in 1.0 release:

  • The bundled Eclipse IDE has been upgraded to 3.4.2 (from 3.4.1 in initial release)

  • The bundled plug-in has been upgraded to 1.0.26 (from version 1.0.24 in initial release)

Limitations and Known Issues

This section lists the limitations of the product and known issues in this release.

Limitations

Some of the features described in the GlassFish Tools Bundle for Eclipse User Guide documentation are available in GlassFish Server v3 Prelude only and not in GlassFish Server v2.1.

Support for Java EE functionality is dependent on GlassFish Server version. Some GlassFish Server features are not available in this release.

Upgrades

An upgrade from an earlier installation is not supported in this release. To install a new version of software, uninstall the previous installation and then install the new version.

Known Issues

Issue 172: Using workspace from previous bundle install produces unexpected results

Description

If you are installing GlassFish Tools Bundle for Eclipse 1.0 version, and select to use a workspace which was created with version 0.9.9 of the bundle, the pre-configured GlassFish v2.1 and v3 Prelude servers are still pointing to your 0.9.9 installation, which can cause errors or unexpected results.

If you create a new workspace to use with 1.0 version, the workspace is configured correctly.

For a complete description of the issue, see Issue 172.

Workaround

The workaround for this issue is to provide the location of the previously installed server and database locations to the new installation. A sample workaround file is available in the Issue 172 page. To apply the workaround:

  • Download the workaround.jar file.

  • Open the jar file.

  • Copy the <expanded-jar-location>/com.sun.enterprise.jst.server directory to your <workspace from previous version>/.metadata/.plugins/ directory.

  • Open the <workspace from previous version>/.metadata/.plugins/com.sun.enterprise.jst.server/installLocation.txt file and edit it to point to the path of your 0.9.9 installation.

    The sample workaround contains the default install path for a Mac OS system, but you can modify the path as applicable for your environment.

  • Start the new version of the bundle.


This issue is fixed in version 1.0 of the bundle. You will be able to reuse the workspaces created with 1.0 version of bundle for later installations without any errors.

What is New

This section provides an overview of the product.

The GlassFish Tools Bundle for Eclipse is an open source project. It is a co-bundle of Eclipse IDE, GlassFishTM Server v2.1 and GlassFish Server v3 Prelude and the plug-in that integrates these products.

The GlassFish Tools Bundle for Eclipse allows you to build applications with the Eclipse IDE and deploy them to the GlassFish Server.

The current release of the bundle includes a new plug-in version 1.0.26, updated from version 1.0.24 included in the initial release. For a chronological list of bug fixes and improvements to the plug-in, see the plug-in release notes.

Platforms

This section provides product availability on platforms.

The product distributions contain Eclipse IDE 3.4.2 Java EE IDE, pre-configured GlassFish Server v2.1 and GlassFish Server v3 Prelude and optionally a JDK (version J2SE 6 1.6.0_12). The Mac OS distribution is not available with JDK option.

If you are selecting a distribution without the JDK, make sure you have a recommended, Sun JDK installed in your environment.

Limitations

This section lists the limitations of the product and known issues in this release.

Limitations

Some of the features described in the GlassFish Tools Bundle for Eclipse User Guide documentation are available in GlassFish Server v3 Prelude only and not in GlassFish Server v2.1.

Support for Java EE functionality is dependent on GlassFish Server version. Some GlassFish Server features are not available in this release.

Upgrades

An upgrade from an earlier installation is not supported in this release. To install a new version of software, uninstall the previous installation and then install the new version.

Fixed Issues

The following issues have been fixed in this release of the bundle:

Issue 172: Using workspace from previous bundle install produces unexpected results

Description

When a 0.9.9 or earlier bundle is installed on your system and a new workspace is created for it, the bundle works as expected and you will not see any issues.

If you are upgrading your bundle later and if the new bundle wants to use the previously created workspace, you will see errors as the bundle is unable to locate the previous GlassFish v2.1 and GlassFish v3 Prelude installations.

For a complete description of the issue, see Issue report.

Workaround

The workaround for this issue is to provide the location of the previously installed server and database locations to the new bundle. A sample workaround is available in the Issue report page. To apply the workaround:

  • Download the workaround.jar file

  • Open the jar file

  • Copy the <expanded-jar-location>/com.sun.enterprise.jst.server directory to your <workspace from previous version>/.metadata/.plugins/ directory.

  • Start the new version of the bundle


This workaround is applicable for 0.9.9 version of the bundle on Mac OS. The sample can be applied for other OS platforms or bundle versions by modifying GlassFish install location in the <expanded -jar-location>/com.sun.enterprise.jst.server/installLocation.txt file.

The issue is fixed in version 1.0 release of the bundle. You will be able to use the workspaces created with this version of bundle for later installations without any issues.

Other issues fixed in 1.0 bundle release:

Issue 179: Domain.xml path not set properly

Description

GlassFish v3 configuration returns null for the domain.xml, in case it already exists. Configuration should find domain.xml path for the existing domain.

For a complete description of the issue, see Issue report.

Issue 178: Reinstall, then new workspace missing post processing

Description

Bundle is installed and a new workspace is created. Then bundle is reinstalled without deleting the previous installation and uses a new workspace. In such a case, post processing, which creates the domain seems to fail.

Issue 177: Server preferences for v2 missing text

Description

When you open the IDE preferences menu, and select GlassFish v2.1 from pre-configured servers, you can not see the server path. Whereas for GlassFish v3 Prelude you can see the server path.

Issue 159: Icons and splash screen issues on windows

Description

The GlassFish bundle uses default icons for installer. It should use GlassFish icons or Eclipse icons.

For a complete description of the issue, see Issue report.

Issue 158: No domain setup in new workspace after first co-bundle run

Description

When the bundle starts using a new workspace, domains do not get created.

For a complete description of the issue, see Issue report.

Issue 141: Redundant “/” in server path when making server configuration

Description

There is a redundant “/” in default server path when running server configuration.

For a complete description of the issue, see Issue report.

Database Schema

The databases, tables, and columns within the MySQL database known as the schema can be manipulated using the web-based phpMyAdmin tool. These operations are typically done by a Database Administrator or a DBA who is familiar with SQL and the relational philosophy of data representation, storage, and querying.

To manipulate the databases, tables, and columns of the MySQL database, after initialization and starting Servers, click on the Launch menu, and select All Applications > Developer Tools > A.M.P. Web App Development > Administer Database. A web-based database management tool called phpMyAdmin opens in a web browser. This tool provides a visual interface to the Data Definition Language (DDL) -- the schema and the Data Manipulation Language (DML), that comprises of inserting, deleting, updating, and selecting or querying aspects of the Structured Query Language (SQL) that MySQL database supports. Using this tool, you can do the Data Definition tasks like creating databases and creating and editing tables for use by your applications. You can also do Data Manipulation tasks like viewing and manipulating data, to create test data, or diagnose problems.

For detailed information about the relational database concepts and SQL, see the <online website (...introduction to relational concepts...) >

* book: chapters 4-12 of _MySQL: The Complete Reference_, by Vikram Vaswani

Customizing AMP Components

After Initialization, you can customize several aspects of the A.M.P. components.

From the Launch menu, select All Applications > Developer Tools > A.M.P. Web App Development > Options.

For example, to start the Servers to automatically at system start up, or each time when you log in, check the appropriate checkbox on the General tab of the Option panel. Only a limited set of options are available in the Options panel. For more advanced options, click the Advanced button on the appropriate tab. Service configuration file for that component will be opened in a text editor, where you can edit the options.

Migrating From Development to Production

The easy User Interface provided from the Launch menu for rapid and easy-to-learn A.M.P. Web App Development is intended for development purposes. We do not recommend running mission-critical websites without customization and tuning for performance, scalability, and security. When you've developed and debugged a working app in this environment, we recommend transferring the database data and PHP files to a production-quality Solaris installation, running on high-availability Sun hardware, that's been carefully tuned to your mission-critical needs.

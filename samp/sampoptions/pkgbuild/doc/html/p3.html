Chapter 2

Getting Started

This chapter describes the basic steps to get started with the developer tools to develop and deploy web applications.

Initializing the Development Environment

Before you can use the Solaris Web application development environment, you must initialize the environment for your login. Every new user logged into the system must initialize their own environment.

To Initialize your Web Application Development environment, click on the Launch menu, and select All Applications > Developer Tools > A.M.P. Web App Development > Initialize.

The first user who initializes must enter the system's root password as the first initialization process installs and configures various operating system services. Subsequent users who initialize skips this step and thus will not be prompted for the root password.

Note - Changes to the Web App Development options will affect all users on the system.

Starting and Stopping Web Stack Components

This section describes the steps to start and stop various Web App Development tools on the Solaris platform.

Note - Before you start the components, you must initialize the development environment for your login account.

Starting and Stopping Apache HTTPd Server

To Start the Servers on your A.M.P. Web App Development environment, click on the Launch menu, and select All Applications > Developer Tools > A.M.P. Web App Development > Start Servers (or Stop Servers).

You can configure the servers to be started automatically at system start up by customizing the start up options. For more information see the section on Customizing.

Testing the Web App Development Environment

To test whether the development environment you have set up is working fine, perform the following simple steps:

To Test the Web App Development Environment

Before You Begin

  • Make sure that you have initialized the environment for you login account.

  • Make sure that the required components are started.

  1. Create a simple web application.

    Alternatively, you can copy the following code into a text file.

    <DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" 
"http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">
<head>
    <meta http-equiv="Content-Type" content="text/html; charset=iso-8859-1" />
    <title>My First Web App</title>
	</head>
<body>
<b>Hello, World!</b>
<p>The date and time are <?php echo date ("r"); ?> - isn't that nice? - isn't that nice?</p>
</body>
>/html>
  2. Save the file to /usr/local/apache2/htdocs/myfirstwebapp.php.
  3. Open a web browser and enter the URL http://localhost/myfirstwebapp.php in the address bar.

    The following message is displayed in the web browser, where the date and time reflects your system's current date and time:

    Hello, world!

    The date and time are Thu, 20 Dec 2007 16:01:07 +0200 - isn't that nice?

Next Steps

Success! Note that the web page contents enclosed within the <?php and ?> delimiters were interpreted by the PHP interpreter at the time the Apache Web Server served up that page. The output of those PHP language statements were substituted into the HTML stream at that location. All other HTML content is served up to the web browser as in static web pages.

Now, reload your browser but make sure that your browser does not use its cache; in most browsers, you can reload the browser by holding down the ALT key while clicking the Reload button. You see the same result, but with a later time. Thus you can see that a PHP page gets executed every time it is displayed (barring local browser caching, which you can suppress through PHP directives), and that you, as a developer, can generate highly dynamic content.
